# Resume Here — 未完成任务索引

> **用途**：记录所有没干完的任务。下次任何对话开新会话，**第一件事就是读这个文件最上面那条**，不需要去翻历史对话。
>
> **规则**：
> - 最新未完成任务放最上面（栈式结构）
> - 任务做完后，**从这里删除**对应条目，并把结果写进变更日志（如有）
> - 用户说「先到这 / 明天继续 / 等会再说 / 停一下 / 我去测试一下 / 后面再测」等中止信号时，Claude 必须**主动**更新此文件，不等用户提醒
> - 「下次起步」要写到能直接复制粘贴给新对话作为 prompt 的程度

---

## 模板（复制下面这段填空）

```md
## YYYY-MM-DD HH:MM — 任务标题（简短）

**Claude Code 会话名：** （/rename 设置的；没设就写"未命名"。建议每开新任务先 /rename，方便重开时在会话列表里搜回来）

**做到哪：**
- （已完成的小步骤，写到具体函数/文件行）

**卡在哪：**
- （技术卡点 / 等用户确认 / 等外部依赖 / 等真机测试）

**触发条件**（长 idle 两步法挂起时必填，普通中止可省略）：
- 条件 1：……（满足任一即可续）
- 条件 2：……

**下次起步：**
1. 第一步具体做什么（含文件路径 + 函数名）
2. 第二步
3. ……

**涉及文件：**
- `path/to/file1`
- `path/to/file2`

**需要用户确认：**
- （如果有，同步登记到同目录的 `pending_questions.md`；如果没有，写"无"）

**最后更新：** YYYY-MM-DD HH:MM
```

**字段说明**：

- **触发条件**：长 idle worker 走两步法时（CLAUDE.md「长 idle worker 处理」段）的 resume 条目**必须**含此段，让未来 revive 时知道为啥这个 worker 之前挂起 + 什么时候该重启。普通任务中止（不是长 idle）可以省略此段。
