# Session Board — 当前活跃对话面板

> **用途**：一张表看清楚现在到底有几个 Claude Code 对话还活着、各自在干啥、最近什么时候动过。
>
> **核心目的**：弥补管家对话的"黑盒滞后"——管家只能看 `/done` 后写到文件的快照，看不到对话中间的实时进度。本文件让每个对话**自己 upsert 自己那一行**，把黑盒变灰盒。
>
> **不是**：会话历史归档。完工的对话从 board 移除，不留痕（完工证据在 git / 变更日志里）。
>
> **规则**：
> - 每个活着的对话占一行，用 `/rename` 的会话名做主键
> - **Session 名 ASCII 单一主键**：Claude `/rename` 名 ≡ tmux session 名 ≡ 本表 Session 字段。小写英文 + 短横线 2-4 词。中文 / 大写不行（tmux 命令行处理非 ASCII 经常炸）
> - **管家 / 派单**禁止编辑 session_board（硬约束）；新对话起来后自注册（CLAUDE.md「会话开头主动注册」段强制）
> - 写入触发：执行 commit / `/catchup` / `/done` / 用户主动 `/rename` 后第一次 Claude 回复
> - 读取触发：`/butler` Step 1 必读；用户随时翻
> - 完工对话（`/done` 走完 + resume_here 没残留）从「活跃对话」段移到「历史活跃」段，**不删除**
> - 「历史活跃」段 **超 500 行触发归档**（不是删除）到 `docs/_archive/session_board/<start>_to_<end>.md`

---

## 状态枚举

| Emoji | 状态 | 含义 |
|---|---|---|
| 🟢 | active | 正在干活，5-10 分钟内有进展 |
| 🟡 | 等用户 | 等用户答 `pending_questions` 或拍板设计 |
| 🔴 | 待收工 | 改完代码但还没 `/done`（容易忘的状态） |
| ⚫ | blocked | 等外部依赖（后端字段 / 真机 / 设计稿 / 账号） |
| ⚪ | 已完工 | 任务收尾完毕，下一轮 upsert 时该删除 |
| ⚪ | 已挂起 | 长 idle 走两步法 kill；触发条件见 resume_here.md，jsonl 留着 `revive` 救活 |
