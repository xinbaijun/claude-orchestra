# Pending Tests — 待测试队列

> **用途**：登记所有"代码已经改了但当场没法测试"的内容（真机 / iOS only / Android only / 长跑 / 需要特殊账号或环境）。
>
> **规则**：
> - 最新待测项放最上面（栈式结构）
> - 完成测试后**直接删除**对应条目，不需要归档（已经测过的事实由 git / commit 见证；本文件只追踪"还没测"）
> - 用户说"我去测试一下 / 我去试试"等中止信号时，Claude 走 `/done` 流程，Step 3 检查本文件
> - **不要**把当场可测的内容登记到这里（"明天再测"不算待测，"得真机才能测"才算）

---

```md
  - 关联任务：（resume_here.md 里哪条 / 对应 commit / 哪天的变更日志条目）
```
