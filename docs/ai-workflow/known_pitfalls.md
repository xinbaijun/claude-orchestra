# Known Pitfalls — 项目踩过的坑

> **用途**：收纳**域特定**、需要排查时才用得上的坑点。CLAUDE.md「常见坑」段只放高频日常会撞的（5-10 条核心），其余沉到这里。
>
> **读者**：主要给 Claude 用 —— 排查问题前先 grep 这里。新会话开头**不需要**默认读这个文档，只在碰到 UI / 原生 / 构建 / SDK 类诡异问题时主动查。
>
> **怎么用**：
> - 想搜 splash / 闪屏问题 → `grep -i splash docs/ai-workflow/known_pitfalls.md`
> - 想搜 iOS 原生问题 → `grep -i ios docs/ai-workflow/known_pitfalls.md`
> - 想搜某个 SDK → 用 SDK 名 grep
>
> **怎么写**：
> - 每条按「现象 → 根因 → 修复」三栏式
> - 标题含可 grep 的关键词（错误信息 / SDK 名 / 文件路径 / 平台名）
> - 修复方案要带具体代码 / 配置 diff，让下次撞到能直接抄
> - 关联 GitHub issue / 文档链接如果有，附上方便交叉验证
> - 修复了/不再相关的坑：**不要删**，挪到底部「已不再相关」段并标日期，保留历史价值
> - 新坑加到对应段的最上面（新的在上）
